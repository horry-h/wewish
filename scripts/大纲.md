# Role / 角色设定
你是一位资深的小程序全栈开发工程师与极简主义 UI/UX 设计师，擅长心理疗愈类产品的交互设计。

# Project Overview / 项目背景
项目名称：【心之解惑 / AI答案之书】
目标人群：面临焦虑、有选择困难症、追求生活仪式感的年轻人。
核心逻辑：用户通过“意念聚焦+长按交互”获取一个随机答案，并可选择由 AI 进行深度心理疏导与解析。

# UI/UX Style / 视觉风格
1. 风格：极简禅意风、深夜模式（Deep Dark Mode）或 莫兰迪疗愈色系。
2. 关键词：呼吸感、留白、流光效果、拟物化书页。
3. 交互：强调震动反馈（Haptic Feedback）与音效的配合。

# Feature List / 功能清单
1. 首页 (Entrance)：
   - 包含“每日一签”入口。
   - 核心区：问题分类选择（情感、事业、学业、财富、通用）。
   - 底部：显著的交互区，提示“闭目默念，长按寻声”。
2. 交互过程 (Interaction)：
   - 用户按住按钮时，屏幕出现书页快速翻动动画或星云汇聚特效。
   - 手机产生高频轻微震动。
   - 伴随纸张翻动或空灵的白噪音。
3. 结果页 (Result)：
   - 核心展示：一句简短有力的“答案”（如：现在就去做、再等等、保持沉默）。
   - AI 解析模块：点击“书灵解读”，展开一段由 AI 生成的温情且富有哲理的文字。
   - 分享功能：生成一张精美长图（支持分享朋友圈、设置微信状态）。

# Deliverables / 请输出以下内容
1. 【设计稿描述】：请用文字详细描述三个主页面的视觉布局。
2. 【交互逻辑】：详细说明“长按-松开-弹出”这一核心动作的技术逻辑。
3. 【微信小程序代码原型】：
   - 提供 WXML 结构（核心组件）。
   - 提供 WXSS 样式（重点是翻页动画和背景流光效果）。
   - 提供 JS 逻辑（包含微信原生震动 API `wx.vibrateShort` 和音效 `wx.createInnerAudioContext` 的调用示例）。
4. 【AI Prompt 策略】：请帮我写一段内置给 AI 的系统提示词（System Prompt），要求 AI 能够结合“用户问题”和“随机答案”生成一段治愈系的解读。

# Constraints / 约束条件
- 交互必须丝滑，避免复杂的层级切换。
- 分享卡片的设计必须符合小红书审美，具备高颜值。
- 考虑到微信小程序的性能，尽量使用 CSS 动画而非重度视频素材。
