## 🔧 技术实现

### 数据结构

```typescript
interface Answer {
  id: number           // 唯一标识
  text: string         // 答案文本 (15字以内)
  tags: string[]       // 分类标签 (支持多标签)
  type: 'positive' | 'neutral' | 'cautious'  // 答案类型
}
```

### 核心函数

#### 1. `getAnswersByCategory(category: string)`
根据分类标签筛选答案
```typescript
export function getAnswersByCategory(category: string): string[] {
  return fullAnswerPool
    .filter(answer => answer.tags.includes(category))
    .map(answer => answer.text)
}
```

#### 2. `getWeightedAnswer(category: string)`
随机获取指定分类的答案
```typescript
export function getWeightedAnswer(category: string): string {
  const answers = getAnswersByCategory(category)
  return answers[Math.floor(Math.random() * answers.length)]
}
```

## 🎯 设计原则

### 1. 简洁性
- 每条答案控制在 **15个字以内**
- 语言精炼，一针见血
- 避免冗长说教

### 2. 启发性
- 提供方向而非答案
- 引导思考而非指令
- 留有解读空间

### 3. 治愈性
- 避免过于负面的词汇
- 传递温暖和希望
- 关注内在成长

### 4. 现代感
- 贴近当代年轻人语境
- 融合网络流行智慧
- 关注心理健康

### 5. 普适性
- 适用于不同人群
- 覆盖多种生活场景
- 兼顾东西方智慧

## 🌟 后续优化方向

### 1. 加权算法
根据答案类型（positive/neutral/cautious）和用户历史，动态调整概率分布。
